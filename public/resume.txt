3. HALAMAN UTAMA (app/page.tsx)
----------------------------------------------------------------

Halaman Home bersifat async Server Component dengan ISR (revalidate: 60 detik).
Data project di-fetch langsung dari Sanity menggunakan query GROQ, diurutkan 
berdasarkan publishedAt (terbaru dulu). Terdiri dari 5 section berurutan:

  [Hero] → [SelectedWork] → [Toolkit] → [Experience] → [Footer]

7. CMS — SANITY SCHEMA (sanity/schemaTypes/project.ts)
----------------------------------------------------------------

Document type: "project" — field yang didefinisikan:

  title             → Judul project (string)
  slug              → URL-friendly slug (auto dari title)
  shortDescription  → Deskripsi singkat (text, 2 rows)
  projectOverview   → Overview panjang (text, 5 rows)
  roles             → Array of string
  publishedAt       → Tanggal publish (datetime)
  categories        → Array of string
  timeline          → Durasi project (string)
  company           → Nama perusahaan (string)
  technologies      → Array of string (tools yang digunakan)
  mainImage         → Thumbnail utama (image + hotspot + alt)
  body              → Rich text (block + inline images + caption)
  showcaseImage1    → Showcase full-width pertama
  showcaseImagesTwoColumn → Max 2 gambar (validasi), layout 2 kolom
  showcaseImageLast → Showcase full-width terakhir
  closingStatement  → Kalimat penutup (text, 3 rows)
  status            → "public" | "private" (visibility control)
